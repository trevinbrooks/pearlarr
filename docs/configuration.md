# Configuration

Every setting Pearlarr reads, group by group: the prose explains how the settings interact; the tables are generated from the config models and are always current.

Pearlarr reads one file, `config.yml`, from its data directory.
`pearlarr paths` prints where that is, `pearlarr config init` writes a starter copy, `pearlarr config validate` checks a filled-in file, and `pearlarr config show` prints the effective configuration with every secret redacted.

## How settings resolve

- A command-line flag beats an environment variable, which beats `config.yml`, which beats the built-in default.
- A blank key (`url:` with nothing after it) takes the built-in default, so the starter file loads as pure defaults.
- An unknown or misspelled key fails validation with an error naming the key; nothing is silently ignored.
- The file holds credentials in plain text. It is created owner-only (`0600`), and a warning fires at load when an existing file is readable by group or other.

## Configuration changes and the cache

Titles that have been handled are cached in `cache.db` and skipped on later runs.
Editing the config does not re-evaluate them: after changing a selection-affecting setting (anything in the `seadex` group, or `imports.languages_*`), run `pearlarr cache remove` so the next run re-checks every title against the new rules.

## qbittorrent

Where grabbed torrents go.
**Leaving any of `host`, `username` or `password` blank puts every run in preview mode:** titles are evaluated and decisions are reported, but nothing is added to qBittorrent, nothing is cached as handled, and no notifications are sent.
Preview mode is the safe way to try a config against live Sonarr/Radarr instances.

<!-- gen:group-qbittorrent - GENERATED by scripts/gen_docs.py from pearlarr/modules/config.py; do not edit between the markers; regenerate: uv run python scripts/gen_docs.py -->
| Key | Default | Values | Description |
| --- | --- | --- | --- |
| `host` | *(blank)* |  | WebUI address of qBittorrent, e.g. `http://localhost:8080`. Leave any of `host`, `username` or `password` blank to run in preview mode: runs report what they would grab, but nothing is added. |
| `username` | *(blank)* |  | WebUI username. Blank switches to preview mode (see `host`). |
| `password` | *(blank)* |  | WebUI password. Blank switches to preview mode (see `host`). |
| `tags` | *(blank)* |  | Tags applied to every torrent added. Blank applies no tags. |
| `options` | *(blank)* |  | Extra keyword arguments passed to the qbittorrent-api client. For example `VERIFY_WEBUI_CERTIFICATE: false` for a WebUI behind a self-signed HTTPS certificate. |

## seadex

How a release is chosen for each title.
Pearlarr takes the SeaDex entry for a title, prefers releases marked best (`want_best`) and dual-audio (`prefer_dual_audio`), then filters by `ignore_tags` and `trackers`, and finally compares the survivors against what the arr already has - by release group name, or by torrent hash when `use_torrent_hash_to_filter` is on.

Private releases are never grabbed (SeaDex carries no download link for them, and no private-tracker auth is supported).
`private_releases` decides what happens when a title's preferred release is private-only: `warn` keeps the title uncached so every run re-checks it; `fallback` takes the best public alternative and remembers that the title was satisfied by a fallback.
Switching from `fallback` back to `warn` re-checks those remembered titles and resurfaces the private-only warning; a fallback substitute never replaces a private copy you already own.

<!-- gen:group-seadex - GENERATED by scripts/gen_docs.py from pearlarr/modules/config.py; do not edit between the markers; regenerate: uv run python scripts/gen_docs.py -->
| Key | Default | Values | Description |
| --- | --- | --- | --- |
| `private_releases` | `warn` | `warn` / `fallback` | What to do when a title's preferred release sits only on private trackers. Private releases are never grabbed: SeaDex carries no download link for them, and no private-tracker auth is supported. `warn`: Warn and skip without caching, so the title is re-checked every run until a public release appears. `fallback`: Grab the entry's best public alternative instead; warn only when none exists at all. |
| `prefer_dual_audio` | `true` |  | Prefer dual-audio releases; when off, prefer Japanese-audio releases. |
| `want_best` | `true` |  | Prefer releases SeaDex marks as best. |
| `ignore_tags` | *(blank)* |  | SeaDex release tags to skip, e.g. `Dolby Vision` or `Deband Required`. Empty skips none. |
| `trackers` | all supported trackers | `Nyaa`, `AnimeTosho`, `AniDex`, `RuTracker`, `AB`, `BeyondHD`, `PassThePopcorn`, `BroadcastTheNet`, `HDBits`, `Blutopia`, `Aither`, `Other`, `OtherPrivate` | Tracker names considered during release selection, case-insensitive. Empty or absent considers every supported tracker. A listed tracker is not necessarily grabbable: private releases are never downloaded, and grabs come only from the public trackers Pearlarr can parse. |
| `ignore_anilist_ids` | *(blank)* |  | AniList IDs never processed. Empty ignores none. |
| `ignore_seadex_update_times` | `false` |  | Re-check entries even when SeaDex has not updated them since the last check. |
| `use_torrent_hash_to_filter` | `false` |  | Match what is already downloaded by torrent hash instead of by release group name. |

## imports

What happens after a torrent is grabbed, on Sonarr runs only (a Radarr run ignores the whole group).
With a `wait_mode` other than `off`, Pearlarr waits for qBittorrent to finish each download, asks Sonarr to rescan, and watches its queue: Sonarr imports the files itself where it can, and Pearlarr steps in with a manual import - pinned to the correct series and episodes by its own mapping - only when Sonarr cannot (a blocked import, or a download Sonarr is not tracking).
Downloads that outlast the run are carried as pending imports and picked up by a later run (`deferred`/`hybrid`), up to `pending_max_age_days`.

<!-- gen:group-imports - GENERATED by scripts/gen_docs.py from pearlarr/modules/config.py; do not edit between the markers; regenerate: uv run python scripts/gen_docs.py -->
| Key | Default | Values | Description |
| --- | --- | --- | --- |
| `wait_mode` | `'off'` | `off` / `deferred` / `blocking` / `hybrid` | When, if ever, to wait for grabbed torrents to finish downloading and then drive Sonarr's import. `off`: Disabled: no waiting, no pending-import records, no manual import. `deferred`: Never block: record grabs and import the already-finished ones on a later run. `blocking`: Block at the end of the run (and on an early break) until downloads finish, then import. `hybrid`: Reconcile deferred imports at run start, then a blocking pass at the end (recommended). |
| `wait_timeout` | `3600` |  | Seconds to wait per torrent for qBittorrent to finish downloading it. |
| `ready_timeout` | `600` |  | Seconds to then wait for Sonarr to rescan and import the finished files. |
| `poll_interval` | `30` |  | Seconds between polls of qBittorrent and the Sonarr queue while waiting. |
| `progress_poll_interval` | `5` |  | Seconds between lightweight wait-screen refreshes (progress bars, speed, ETA) within each poll interval. `0` disables the extra refreshes; rows then advance once per `poll_interval` while spinners and timers still animate. A value at or above `poll_interval` behaves as disabled. |
| `mode` | `auto` | `auto` / `move` / `copy` | How the import takes finished files into the library. `auto` lets Sonarr choose between moving and copying (a still-seeding torrent is copied); `move` and `copy` force it. Forwarded to Sonarr verbatim. |
| `post_import_category` | *(blank)* |  | qBittorrent category applied to a torrent once its import is verified complete. Useful to give finished torrents different seeding rules. Blank keeps the add-time category. Only applies when `wait_mode` is not `off`. |
| `default_quality` | *(blank)* |  | Sonarr quality name, e.g. `Bluray-2160p`, filling the gaps when a file's quality cannot be detected. Useful on a 4K instance. Blank adds no default; a name matching no Sonarr quality definition is warned about and ignored. |
| `languages_dual` | `[Japanese, English]` |  | Languages tagged on files imported from dual-audio releases. An empty list falls back to the default; a file is never imported with no language (Sonarr would read it as Unknown and could re-grab it). |
| `languages_single` | `[Japanese]` |  | Languages tagged on files imported from single-audio releases. Empty falls back like `languages_dual`. |
| `pending_max_age_days` | `14` |  | Days a pending import may wait for its download before its record is dropped. |
| `digest_interval` | `300` |  | Target seconds between wait-progress digest lines on a non-interactive console (Docker, cron). |

## schedule

Cadence for scheduled mode (`pearlarr run scheduled`, the default command): one cycle runs every configured arr, and the process then sleeps for `interval_hours`.
The config is re-read each cycle, so edits take effect without a restart.
In Docker the schedule comes from `PEARLARR_CRON` instead.

## advanced

Rate limiting, caching, run caps, and logging.
`detect_arr_activity` re-checks titles whose files changed arr-side (an upgrade or manual replacement in Sonarr/Radarr) even though they are cached as handled; turn it off if a release you deliberately replaced keeps being re-grabbed.
The console format and level apply to terminal output; a plain-text file log is always written to the data directory regardless.

<!-- gen:group-advanced - GENERATED by scripts/gen_docs.py from pearlarr/modules/config.py; do not edit between the markers; regenerate: uv run python scripts/gen_docs.py -->
| Key | Default | Values | Description |
| --- | --- | --- | --- |
| `sleep_time` | `2` |  | Seconds slept between API queries, as rate limiting. `0` disables the sleep. |
| `cache_time` | `1` |  | Days downloaded mapping sources are kept before re-downloading. `0` re-downloads every run. |
| `interactive` | `false` |  | Prompt for a choice when several torrents match, instead of taking the best automatically. |
| `max_torrents_to_add` | *(blank)* |  | Cap on torrents added per run. Blank means unlimited. |
| `detect_arr_activity` | `true` |  | Re-check titles whose files Sonarr or Radarr changed since the last run. Polls each arr's history at run start. Turn off if a release you replaced arr-side keeps being re-grabbed. |
| `log_level` | `INFO` | `DEBUG` / `INFO` / `WARNING` / `ERROR` / `CRITICAL` | Minimum level written to the console and the file log, case-insensitive. |
| `log_format` | `auto` | `auto` / `rich` / `plain` / `json` | Console output format. `auto` resolves to `rich` on a terminal and `plain` when piped or under Docker; `plain` and `json` also disable the live progress views. |

## mappings

Where the AniList-to-arr ID and episode mappings come from.
`anibridge_mappings` is the primary source; the other two fill gaps (AniDB mainly for specials).
Leave all three blank: sources are auto-downloaded and cached for `advanced.cache_time` days.
Setting one to `false` disables it; `anime_mappings` and `anibridge_mappings` also accept an inline mapping for power users.

<!-- gen:group-mappings - GENERATED by scripts/gen_docs.py from pearlarr/modules/config.py; do not edit between the markers; regenerate: uv run python scripts/gen_docs.py -->
| Key | Default | Values | Description |
| --- | --- | --- | --- |
| `anime_mappings` | *(blank)* |  | Kometa `Anime-IDs` source, a fallback for IDs the primary graph misses. Blank auto-downloads it; `false` disables it; an inline mapping is accepted. |
| `anidb_mappings` | *(blank)* |  | AniDB `anime-lists` XML source, a fallback used for specials. Blank auto-downloads it; `false` disables it. |
| `anibridge_mappings` | *(blank)* |  | `anibridge-mappings` v3 graph, the primary source of IDs and per-season episode maps. Blank auto-downloads it; `false` disables it; an inline mapping is accepted. |
